# 秋招助手 · 静态展示版

这是「秋招助手」的**纯前端只读演示**，无需后端即可运行，用于展示“JD 岗位画像 → 针对性问题 → 证据诊断 → 训练行动 → 下一场验证 → 长期记忆”的完整路径。

- 面试复盘：展示训练分、证据覆盖度、六项 PM 能力、子维度评分、缺口、追问和训练验收
- 阶段成长报告：展示跨场重复缺口、能力变化、长期训练、评分版本可比性和 outcome 样本边界；当前演示明确保持样本不足，不伪造结果比较；单场短期记忆单独呈现
- 训练验证：展示已改善、部分改善和尚未验证三档状态；状态来自预置演示数据，不代表真实面试已经改善
- 面后结果反馈：面试档案顶部展示“等待结果 / 通过 / 未通过”的自报入口和状态说明；静态 Demo 只展示预置状态，不保存真实反馈
- 面经资料库：展示 Agent 查询轨迹、公开页自动读取尝试、规则相关度、匹配理由、可信度状态和来源状态
- 公开搜索与 Agent 调研分开演示：两者都展示 JD 驱动的少量查询和限定平台的公开页读取尝试；Agent 只负责查询规划，相关度由规则计算，不依赖额外示例面经
- 公开面经候选是合成演示数据，明确标为待确认，不对应可验证原帖；数据在 `demo_data.js`
- 真实后端会自动尝试读取限定平台公开 HTML 并回填可见文本，但不会登录、读 Cookie、读取私密内容或绕过访问控制
- 静态版的查询、JD 解析、Agent、成长报告按钮只返回带 `demo_only` 标记的预置只读数据，不联网、不调用模型、不写入资料库；保存、AI 复盘、上传、训练验证等其他写操作会提示「需在完整版体验」
- 输入区的“可能包含敏感信息”提示只做手机号、邮箱、身份证号的 best-effort 检测，不阻止操作，也不代表完整匿名化；静态版不会发送这些输入

完整版（带实时 AI 生成、数据存储）见项目主仓库的 `app/`，可本地运行或部署到带后端的平台。
