# 秋招助手 · Autumn PM Coach

面向中文产品经理的**JD 主线面试准备、复盘与长期训练工作台**。

它不试图替你完成一场实时面试，而是处理面试结束后最容易被忽略的一段时间：趁记忆还清晰，把 JD、面试问题、自己的回答和外部面经整理成可复盘的证据，再把一次表现转化成下一次可以验证的训练动作。

> **JD 画像 → 针对性问题 → 面经研究 → 面后记录 → JD 覆盖诊断 → 训练验证 → 长期成长**

在线可操作版：[Render](https://autumn-pm-coach.onrender.com/) · 在线只读 Demo：[Hugging Face Space](https://huggingface.co/spaces/Vergessne/autumn-pm-coach) · 本地实操地址：[http://127.0.0.1:8765](http://127.0.0.1:8765)

Render 版本需要输入部署者配置的访问口令；Hugging Face 版本只用于浏览预置演示数据，不保存真实记录。

## 为什么做

很多面试复盘最后只剩下两句话：“感觉还可以”和“下次好好准备”。问题不在于没有复盘，而在于复盘缺少一个能持续积累的判断过程：

- 面试刚结束时，最有价值的细节往往还没有被记录下来；
- 只看结果好坏，很难知道真正薄弱的是指标、取舍、推进，还是表达；
- 只看一场面试，容易把偶然表现误认为稳定能力；
- 网上面经可以帮助准备问题，但搜索结果不等于事实，也不应该直接替代个人经历。

所以这个项目把求职过程拆成一条可以回看的训练链：

1. **先理解岗位**：把 JD 变成岗位画像，判断可能被问什么、哪些要求需要准备证据。
2. **还原这一场**：记录面试官问了什么、自己怎么答、哪里被追问、哪些结果没有讲清。
3. **解释为什么**：把回答中的原文证据对应到 JD 要求、PM 能力和可观察缺口，而不是只给一个分数。
4. **决定下一步**：把缺口变成训练行动，并在后续兼容面试中自动检查是否出现了可观察改善。

## 整体思路

产品由一条 JD 主线训练闭环组成，公开研究提供准备线索，长期记忆提供跨场时间维度：

```text
岗位 JD / 简历 / 公开面经
          │
          ▼
  JD 岗位画像
  生成岗位重点、搜索线索和面后速记问题
          │
          ▼
  面试转写 / 手写记录 / 面后补充复盘
          │
          ▼
  JD 覆盖 + 证据型诊断
  岗位要求 → 原文证据 → PM 能力 → 评分理由 → 能力缺口
          │
          ▼
  短期训练
  下一条追问 → 训练动作 → 验收标准 → 下一场验证
          │
          ▼
  长期记忆
  重复缺口、训练验证、能力趋势、开放行动、结果反馈
```

这里有一个重要的判断边界：**模型负责理解和归纳，系统负责把结论落回证据和可执行动作。** 搜索资料用于帮助准备问题，不会被冒充为候选人的面试事实；训练分用于决定先练什么，不代表录用概率。

## 一次面试如何变成训练

### 1. 先从 JD 建立考察上下文

粘贴 JD 后，系统会提取岗位职责、任职要求、关键词、可能追问和搜索主题。它不只是帮你“搜面经”，还会把这些要求带入后面的速记问卷：

- JD 强调指标设计，会优先追问指标口径、归因和验证；
- JD 强调跨团队推进，会优先追问分歧、取舍、对齐和落地；
- JD 强调用户问题和业务理解，会优先追问目标、场景、优先级和业务结果。

即使暂时没有关联简历，也会根据 JD 生成针对性的命中题和补刀题，而不是只给泛泛的“表现如何”。

### 2. 用面经做准备，不用面经替你作答

研究 Agent 会根据公司、岗位、轮次和 JD 主题生成少量搜索路径，并尝试发现公开的全网、小红书或牛客资料。Agent 只负责安排查询词，候选相关度由后端固定规则计算，因此不需要用户准备或寻找额外示例面经。资料的作用是帮助你提前知道可能出现的追问方向，例如：

- 这个岗位常见的项目深挖点是什么；
- 面试官可能如何继续追问指标和结果；
- 同一类岗位通常关注哪些协作或业务问题。

搜索候选、自动读取、AI 预审和人工确认是不同阶段。外部资料只进入“问题准备上下文”，不会混入本场面试的表现证据。

相关度只回答“这条候选值不值得打开”，页面会直接显示命中的公司、岗位、轮次或主题；没有填写的维度不参与扣分。它和原帖是否真实、是否可以引用是两件事：后者仍由公开页读取状态、原文摘录和人工确认决定。

### 3. 结合 JD 与简历直接推理发问

速记问卷不是固定问题库，而是根据目标岗位和个人经历推理这场面试最可能追问什么：哪些项目会被深挖、哪些能力在简历里证据不足、哪些回答需要补充指标、取舍或业务解释。公开面经只作为额外的提问线索，帮助你理解问题为什么出现、面试官可能在判断什么。

因此生成的问题会分成“命中”和“补刀”：前者核对 JD 与经历的交集，后者补足岗位要求与已有证据之间的空白，并标注对应的 PM 能力。你回答问题后，可以继续补充面试官原话、自己的决策、没有接住的追问，以及没有讲完整的结果、数字或归因。

重点不是把面试整理成一篇完整纪要，而是通过一组有上下文的问题，把当时的思考过程重新讲清楚。

### 4. 诊断关注能力缺口，而不是制造一个漂亮分数

PM Coach 保留六项能力：

| 能力 | 关注的问题 |
| --- | --- |
| 产品判断 | 是否定义了用户问题、目标和优先级，能否解释方案取舍 |
| 项目主导力 | 是否说清个人职责、关键决策、结果和复盘 |
| 指标与实验 | 是否定义指标、拆解链路、解释归因并给出验证方法 |
| 推进与协作 | 是否处理分歧、约束、风险和落地闭环 |
| 结构化表达 | 是否结论先行、层次清楚、能应对连续追问 |
| 业务与岗位理解 | 是否把个人经历连接到 JD、业务目标和岗位场景 |

每项能力还会拆成可观察的子维度。系统会区分“有原文证据”“缺少证据”“证据矛盾”和“不适用”，再给出评分理由、置信度和优先缺口。这样低分不只是结论，而是下一次应该补哪一段回答的提示。

### 5. 训练动作必须能够被验收

每个重点缺口都会留下短期训练记忆，例如：

> 用“核心指标 + 护栏指标 + 归因假设 + 验证方法”重写一次项目成功标准。

它还会生成下一条追问和验收标准。训练不是“多准备一些”，而是要能判断自己是否已经补上：是否说清口径、是否给出验证、是否能解释结果变化。

### 6. 下一场会自动验证训练是否见效

每个训练行动会绑定来源缺口和能力。完成后续复盘时，系统只在满足同岗位族、日期更晚、公司或岗位相同的条件下自动寻找验证场，并检查验证场是否真正覆盖能力、原 gap 是否再次出现，以及同一能力的 exact score 是否提升至少 0.5。

结果只有三档：**已改善、部分改善、尚未验证**。这不是招聘结论，也不会修改原场评分；验证证据只来自后续面试的归一化原文证据。

### 7. 多场之后才讨论成长趋势

单场复盘回答“这一次哪里需要改”，长期记忆回答“这个问题是否反复出现”。系统会聚合已复盘场次中的重复缺口、能力变化、未完成行动和面试结果反馈，并在样本不足时保持克制，不把一次偶然表现扩展成长期判断。

## 怎么使用

1. 新建面试，填写公司、岗位、轮次和 JD。
2. 点击“解析 JD 并搜索”，生成统一岗位画像并查看研究线索。
3. 点击“生成速记问卷”，按能力和 JD 记录刚结束的面试。
4. 把问卷答案并入转写，或直接在统一记录框中补充面试细节。
5. 点击“生成 PM 复盘”，查看 JD 覆盖、证据、评分理由、缺口和训练验收。
6. 完成下一场兼容面试后，回到成长报告查看训练验证和重复问题。
7. 收到面试结果后回到档案中补填结果；它只作为描述性训练反馈。

## 两个入口

### 静态 Demo

`static_demo/` 是纯前端只读展示版，用合成数据展示完整产品逻辑，适合直接在线浏览或部署到 Hugging Face Static Space。

部署说明：[DEPLOY_HUGGINGFACE.md](./DEPLOY_HUGGINGFACE.md)

### Render 在线版

Render 版本运行完整后端，可以实际保存面试档案并调用 Gemini。首次进入时输入部署者设置的访问口令；进入工作台后，依次填写 JD、面试记录，再使用 JD 解析、速记问卷和 PM 复盘。

部署时至少需要配置：

```text
APP_DEMO_MODE=0
APP_ACCESS_TOKEN=<部署者设置的访问口令>
GEMINI_API_KEY=<自己的 Gemini API Key>
GEMINI_MODEL=gemini-3.1-flash-lite
```

访问口令只负责进入工作台，`GEMINI_API_KEY` 才负责模型调用。不要把 API Key 写入 README、代码仓库或公开截图。

### 本地实操版

本地版支持保存自己的面试记录、解析 JD、调用研究 Agent、生成真实复盘、记录训练动作和查看成长报告。数据默认保存在本机，模型调用由你在本地配置。

```bash
cp app/.env.example app/.env
# 编辑 app/.env，至少填写 GEMINI_API_KEY
python3 -B app/web_app.py --host 127.0.0.1 --port 8765
```

打开 [http://127.0.0.1:8765](http://127.0.0.1:8765)，可以先点击“载入演示档案”查看完整诊断，再点击“新建面试复盘”录入自己的资料。

页面会提供可能包含敏感信息提示，但它不替代你对输入内容的人工检查。
