# 秋招助手 · Autumn PM Coach

面向中文产品经理的**面试后复盘与长期训练工具**。它把 JD、简历、面试转写和公开面经整理成一条可回看的训练链路：

`JD 拆解 → 研究 Agent → 面后诊断 → 短期训练 → 长期记忆`

它不提供实时面试辅助，也不把训练分解释成录用概率。

## 30 秒看懂

| 输入 | 本场诊断 | 长期记忆 |
| --- | --- | --- |
| JD、简历、面后速记 / 转写 | 六项 PM 能力、原文证据、评分理由、能力缺口、追问和训练验收 | 重复缺口、能力趋势、开放行动、结果反馈和数据边界 |

## 怎么用

1. 新建面试，填写公司、岗位、轮次和 JD。
2. 点击“解析 JD 并搜索”，自动生成岗位画像、搜索主题和面经候选。
3. 点击“生成速记问卷”，结合 JD、简历和研究线索记录刚结束的面试。
4. 点击“生成 PM 复盘”，查看六项能力和证据型训练建议。
5. 在“成长报告”查看跨场重复缺口和下一阶段训练。
6. 收到面试结果后，在“面试档案”顶部回填“等待结果 / 通过 / 未通过”。

结果反馈使用 `outcome_source=self_reported` 进入长期记忆，只用于训练样本统计，不改变本场评分，也不代表录用判断。样本不足时页面会明确显示，不会强行比较。

## 核心能力

### PM Coach V2

保留六项能力：产品判断、项目主导力、指标与实验、推进与协作、结构化表达、业务与岗位理解。每项拆成四个可观察子维度，后端根据固定权重计算 `exact_score`，再映射到 1–5 分。

诊断只接受面试转写或速记中的可核对证据；没有证据就标记为缺失或未验证，并降低置信度。训练分只表示练习优先级。

### JD 驱动研究 Agent

解析 JD 后，Agent 自动生成公司、岗位、轮次、主题和同义词查询，搜索全网、小红书或牛客公开候选，并尝试读取具体公开原帖。研究线索用于生成问题，不会冒充面试事实；复盘仍只使用本场原文证据。

小红书只走公开网页搜索和公开页面读取，不登录、不读取 Cookie、不抓取私密内容。自动读取的原帖仍会带来源状态，只有人工确认后才可作为事实型引用。

### 长短期记忆

- 短期记忆：本场优先能力、下一条追问、训练题和验收标准。
- 长期记忆：跨场重复缺口、能力变化、未完成行动、结果反馈和来源时间线。
- 数字和计数由本地确定性聚合生成，数据不足会降级说明。

## 两个入口

### 静态 Demo

`static_demo/` 是纯前端只读展示版，适合公开展示或部署到 Hugging Face Static Space：[在线 Demo](https://huggingface.co/spaces/Vergessne/autumn-pm-coach)。它使用合成演示数据，不联网、不调用模型、不保存真实输入。

部署说明见 [DEPLOY_HUGGINGFACE.md](./DEPLOY_HUGGINGFACE.md)。

### 本地实操版

完整版位于 `app/`，支持本地 JSON 保存、JD 解析、研究 Agent、真实复盘、训练行动和成长报告。需要配置模型 API Key，数据默认只写入本机 `app/data/`。

```bash
cp app/.env.example app/.env
# 编辑 app/.env，至少填写 GEMINI_API_KEY
python3 -B app/web_app.py --host 127.0.0.1 --port 8765
```

打开 [http://127.0.0.1:8765](http://127.0.0.1:8765) 即可实际操作。点击“载入演示档案”可以先查看完整诊断，再点击“新建面试复盘”录入自己的资料。

本地回环地址默认免登录；部署到公网必须设置 `APP_ACCESS_TOKEN`。本项目不提供多用户隔离或生产级队列。

## 数据边界

- 本地记录、简历和面经默认保存在 `app/data/`，该目录已被 Git 忽略。
- 点击复盘、JD 搜索、速记出题或转写时，相关内容会发送给你配置的模型或搜索服务。
- 页面会提示可能包含手机号、邮箱或身份证号，但这不是完整脱敏。
- 页面提供“可能包含敏感信息提示”，此提示不代表完整脱敏。
- 静态 Demo 只展示预置数据，不发送真实资料。
- 小红书候选不等于事实；自动读取、AI 预审和人工确认是不同状态。
- outcome 和训练分不用于推断录用概率。
