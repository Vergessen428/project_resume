# 运行态验收记录

## 范围

本记录区分三种验收：静态资源可加载、浏览器中的演示交互、动态后端的真实任务链路。三者不能互相替代。

## 2026-07-14 本机浏览器检查

使用本机临时静态服务器和 in-app browser 完成以下路径：

- 载入演示档案后，复盘页显示六项 PM 能力、子维度、证据覆盖度、置信度、追问和训练验收。
- 进入面经资料库，选择小红书后，候选卡显示平台限定查询、相关度分解、`manual_check_required` 和待人工确认状态。
- 点击“Agent 自动调研并收录”后，页面显示搜索、公开页读取和停止原因轨迹；静态环境明确显示未联网、未写入资料库。
- 返回面试复盘生成速记问卷，问题中带有 JD 和公开线索来源提示；这些线索不进入本场事实证据。
- 在 390 x 844 视口下检查首页、复盘和研究交互，页面无横向溢出，长文本未越出视口。

## 可重复的命令验收

```bash
python3 -B scripts/static_demo_http_smoke.py
python3 -B -m unittest discover -s tests -p 'test_*.py'
```

命令验收覆盖静态入口、同源 CSS/JS、404、演示数据和前端契约；它不能模拟浏览器点击，也不能证明真实搜索已经联网。

GitHub Actions 已配置独立的 `browser-smoke` job：固定 Node 20、Playwright 1.52.0 和 Chromium，运行
`scripts/browser_demo_smoke.mjs`，覆盖复盘、公开小红书候选、Agent 轨迹、JD 到速记问题、只读写操作和移动端溢出，并上传桌面/移动截图。首次 CI 运行通过前，这条验收仍不能视为完成。

## 当前限制

- 静态 Demo 的 JD 解析、搜索、Agent、成长报告和写操作均是 `demo_only` 预置响应。
- 只有动态后端的 `/api/research/agent` 才会按 JD 生成查询、读取白名单公开 HTML、持久化候选并记录任务状态。
- 小红书公开页能被读取，不等于原帖身份、发布时间、正文真实性已经确认；`approved` 仍需要人工确认。
- 浏览器 CI 已配置但尚未在当前工作树中取得一次绿色运行记录；本机检查不能替代 CI 结果。

## 进入真实可用阶段的补充条件

1. 在 CI 中固定浏览器版本，覆盖 JD → Agent → 速记问题、资料库状态门和移动端截图差异。
2. 为真实公开搜索增加脱敏 query-document 标注，持续记录 Precision@K、Recall@K、NDCG@K 和来源状态混淆矩阵。
3. 为公网后端补齐用户隔离、持久化数据库、密钥轮换、任务队列和日志脱敏后，才允许承载真实个人资料。

非回环绑定在非 Demo 模式下必须配置 `APP_ACCESS_TOKEN`；缺少口令时服务拒绝启动，不能把本机免密策略误带到公网。
