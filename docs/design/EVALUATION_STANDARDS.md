# 评测标准

这份标准把“搜索相关性”“资料可信度”和“面试表现评分”拆成三个不同问题，避免一个分数承担不该承担的判断。

## 1. 搜索相关性

相关性回答的是：**这个搜索结果是否值得候选人打开，来解决当前的信息需求？**

信息需求由公司、岗位、轮次和主题组成。搜索摘要初筛使用固定权重：

| 维度 | 权重 | 判断问题 |
| --- | ---: | --- |
| 公司匹配 | 30% | 是否与目标公司或明确的公司上下文相关？ |
| 岗位匹配 | 25% | 是否与目标岗位/职能相关？ |
| 轮次匹配 | 15% | 是否与目标轮次或面试阶段相关？ |
| 主题匹配 | 15% | 是否涉及当前要查的 case、指标、项目等主题？ |
| 面试具体性 | 10% | 是否像真实面试经历，而不是泛泛求职建议？ |
| 时效性 | 5% | 日期是否仍有参考价值？ |

最终相关度是各项 0–100 分按权重加权后的 0–100 分。它只决定“是否值得人工打开”，不代表原帖真实、准确或可引用。

只有人工摘录至少 80 个字符的原帖正文后，系统才进入可信度预审。`confidence` 是可信度量纲，`relevance` 是相关性量纲，二者不能互换。

这个设计沿用信息检索的基本原则：相关性要针对用户的信息需求，而不是单纯看查询词是否出现在文档里；离线评测还需要查询集合、文档集合和人工 relevance judgments。参见 [Stanford Introduction to Information Retrieval](https://nlp.stanford.edu/IR-book/html/htmledition/information-retrieval-system-evaluation-1.html) 和 [NIST TREC relevance judgments](https://trec.nist.gov/data/reljudge_eng.html)。

## 2. PM 面试评分

评分对象是一次面试记录中的**可观察行为**，不是候选人的人格或最终录用概率。项目使用六个维度：

- 产品判断：用户问题、目标、方案取舍和优先级
- 项目主导力：个人职责、关键动作、结果和复盘
- 指标与实验：指标口径、拆解、归因、验证和量化结果
- 推进与协作：分歧、资源约束、风险取舍和闭环
- 结构化表达：结论先行、层次、追问应对和简洁度
- 业务与岗位理解：个人经历与 JD、公司业务、岗位场景的连接

每个维度使用 1–5 行为锚点：

| 分数 | 标准 |
| ---: | --- |
| 1 | 没有展示相关行为，或回答与问题明显不匹配 |
| 2 | 提到了相关点，但缺少关键定义、个人动作、取舍或结果 |
| 3 | 基本合格，有例子和判断，但仍有一处重要环节不完整 |
| 4 | 行为具体，能解释取舍、协作和结果，仅有少量细节缺口 |
| 5 | 证据闭环，能连接用户/业务、决策、指标、结果和复盘 |

每个结论必须先在转写中找到原文证据。证据不存在时，结论应省略或标记“无转写原文可佐证”；记录太短时减少输出项，不用默认 3 分填满报告。

这套维度与公开 PM 招聘材料中常见的 product thinking、problem solving、metrics/data、collaboration/communication、execution/ownership 等能力重叠，但分数只用于个人复盘和练习优先级，不作为招聘决策。参考 [Product-Led Alliance 的 PM scorecard](https://www.productledalliance.com/content/files/2026/01/Hiring-a-product-manager_-Interview-questions.pdf)。

## 3. 可复算的质量指标

- 搜索候选：记录每个候选的相关度分解、初筛理由和真实 URL。
- 资料库：记录原文摘录、可信度、限制项和人工确认状态。
- 单场复盘：记录每条结论的证据、评分锚点和下一步练习。
- 跨场趋势：只统计已经完成复盘的记录；样本不足时显示数据不足，不生成确定性结论。

## 4. 子维度和证据覆盖度

六项 PM 能力各包含四个子维度，固定权重见 `skills/pm-interview-evaluation/references/pm-rubric.md`。后端只对适用且有可核对原文证据的子维度计入 `exact_score`；非法 ID 被丢弃，分数被限制在 1–5。

`evidence_coverage` 是有证据子维度权重 / 适用子维度权重。覆盖度达到 0.75 且至少有两条有效证据时，单项诊断才可达到高置信度；没有转写证据时必须是低置信度。`coach_score` 是六项 `exact_score` 的训练信号映射到 0–100，不是招聘评分。

## 5. 训练验收

每个重点缺口需要给出一个练习动作、一道重写题、最多四条可观察成功标准和一条追问。通过验收的含义是下一次回答满足这些标准，而不是模型分数自动提高。跨场聚合只使用已完成复盘中的规范化 `canonical_gap_id`。
