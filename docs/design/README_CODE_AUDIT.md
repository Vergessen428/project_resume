# README 与代码核对记录

## 当前已实现

- JD 解析结果会进入研究主题、查询同义词和面后速记问题。
- 研究 Agent 会规划查询、去重候选、尝试读取限定平台公开 HTML，并返回查询轨迹、读取状态和停止原因；手动公开搜索入口也会做一次有边界的读取，但不改变来源可信度。
- 小红书只走公开搜索和白名单页面读取，不使用登录态、Cookie 或私密接口。
- 六项 PM 能力各有四个固定权重子维度，后端验证证据、计算 `exact_score` 并映射到 1–5；未覆盖的能力显式返回 `score: null`，新模型输出不会回退到未加权总分。
- 复盘输出包含证据覆盖度、置信度、缺口、训练题和下一条追问。
- 训练行动有稳定身份；重新生成复盘时，存储层合并并保留已有训练尝试、验收状态和完成时间。
- 长期记忆使用确定性统计，模型只负责解释跨场模式。
- 阶段成长报告的结构化字段由长期记忆投影，`report_grounding` 记录版本和输入边界，模型解释不能新增 pattern、次数或训练行动。
- `quality_gold_cases.json` 保存版本化人工标注约束，回归模型输出的分数范围、缺口、证据和训练行动；`relevance_cases.json` 现在包含 6 条带数据集版本的脱敏相关性标注，覆盖平台约束、岗位/主题同义词、缺失轮次、轮次匹配和时效性权重边界，不把坏输出或低相关候选静默丢弃。
- `evaluate_all_fixtures()` 和 `python3 -B -m app.core.evaluation_harness` 提供统一离线回放入口，覆盖归一化、质量黄金样例、研究 Agent 安全和相关性排序四类检查。

## 本轮可信度闭环

- 复盘 schema 为 `2.1`，记录 `scored_by`：实际 provider、model、prompt/rubric 版本和评分时间。
- 旧数据缺少评分元数据时保留为 `legacy_unknown`，不改变历史分数。
- 面试记录支持显式来源为 `self_reported` 的可选 `outcome`，长期记忆按 4 场描述性、6 场双组样本门槛输出结果反馈；旧记录缺少来源字段时不参与聚合。
- 混合模型或旧数据会标记为不可直接比较。
- 公开页每一跳重定向都会先校验白名单和平台路由；非白名单或跨平台目标会在跟随前被拒绝，自动读取仍不会自动变成可信资料。
- 资料进入 `approved` 时会记录人工确认模式、确认时间和正文长度；编辑、清除或重新预审会清除旧 provenance，历史缺少该字段的记录仍按兼容规则读取但不补猜确认来源。
- 平台首页或泛入口不会被当作原帖候选；候选、重定向和人工确认都要求具体公开帖子路径。
- 手动录入和更新资料也复用同一公开域名白名单，不能通过粘贴任意站点摘录绕过来源边界。
- JD 驱动的 Agent 会由后端自动收录候选；获取到足够公开正文时自动触发 AI 预审，仍保留 `auto_fetched_unverified` 来源标记和人工确认入口。浏览器中断不会丢失已收集候选。
- 同一 URL 先前只有脚本壳时，后续获得足够正文会补齐空候选并重新预审；已有人工摘录不会被自动读取结果覆盖。
- Agent 会按查询计划复用短时本地候选缓存，避免重复解析 JD 时重复联网；缓存命中不会绕过现有来源状态和可用性闸门。
- 面试保存时会保留 JD 研究快照中的查询、相关度分解、来源状态和 `question_leads`，但不复制研究原帖正文；刷新后仍能用这些线索生成速记题和复盘上下文。
- 研究快照同时保存来源记录 ID；出题和复盘前会重新检查当前资料状态。资料被删除或降级后仍可作为历史线索显示，但不会继续标记为 `citation_allowed`。
- 研究资料的“已确认”不能由客户端快照自行声明，必须绑定 `research_id` 并通过当前资料库门槛。
- Agent 核心循环和候选收录都会强制公开域名白名单；AI 的预审建议还要经过正文长度、置信度和状态枚举的确定性门槛。
- 相关度只接受六个受控子维度并由后端固定权重重算；模型单独返回的聚合 `relevance` 或缺少分解的初筛结果不会进入排序。
- 训练行动项支持验收状态、时间和备注；长期记忆支持来源展开、缺口重命名/忽略；数据页支持导出、备份/恢复和选择性清理；保留策略支持预览、显式确认和只清理原始正文。
- 静态 Demo 的预置查询、JD 解析、Agent 和成长报告操作均带 `demo_only`，不联网、不调用模型、不写入资料库；预置小红书候选明确为合成演示数据。
- 备份默认保留最近 7 份已校验副本，不自动删除损坏/旧版未校验副本；恢复回滚不完整时写入恢复标记，启动与健康接口会失败并给出恢复边界。
- HTTP 返回不会暴露备份文件或恢复问题的本机绝对路径；`/api/*` 默认按来源 IP 限流，健康探针不计入。
- 后台任务状态写入 `tasks.json`，运维事件写入字段白名单的 `operations.jsonl`；任务日志不含原始输入、结果或 prompt，日志会轮换，重启中的任务标记为 `abandoned`；排队任务可取消，运行中任务采用协作式取消并丢弃晚到结果。
- 缺口标签治理事件单独保存在 `memory_override_events.json`，并随导出/备份恢复；事件不包含转写正文。
- 成长页可通过带确认的撤销接口恢复单次治理操作之前的标签状态；撤销事件继续写入审计历史，且不修改面试原文或评分。

## 仍然明确不是的能力

- 不是录用概率模型，也不根据 outcome 反推评分因果。
- 不是小红书登录态爬虫，无法保证读取动态脚本页面的正文。
- 不是多用户 SaaS；本地 JSON 只适合单用户、小数据和单实例。
- 静态 Demo 不是实时后端，所有写操作和联网操作都是只读演示。

## 严格审计后仍需解决的问题（2026-07-14）

这些不是页面缺字段，而是距离真实长期使用仍有差距的工程问题：

1. **公开页读取不等于原帖真实性**：Agent 会自动发现、打开白名单公开 URL，并在正文足够时触发 AI 预审；`auto_approved` 只是“满足确定性门槛 + 模型预审”的问题线索，不具备事实引用资格，也不证明作者、发布时间、内容没有被转载或截断。只有人工确认后的 `approved` 才能进入 `approved_context`。小红书脚本壳、地区限制、登录墙和重定向仍会降级为 `manual_check_required`。
2. **搜索排序还没有足够人工标注**：当前离线相关性 fixture 只有小规模脱敏样例，回放宏平均 Precision@K 仍是校准信号，不是搜索质量证明。继续增加公司、岗位、轮次、主题和平台的 query-document judgments 后，才能调整阈值和 Agent 停止策略。
3. **本地运行边界仍然存在**：JSON store、进程内任务池和 IP 限流只适合单用户/单实例。公网 Render 版本在没有正式账号隔离、密钥托管、持久卷、分布式任务队列和监控前，不应承载多人真实面试资料。
4. **浏览器验收不能只靠源代码**：GitHub Actions 已加入标准库 HTTP smoke，并配置独立的 `browser-smoke` job，固定 Node 20、Playwright 1.52.0 和 Chromium，验证静态 Demo 的复盘、XHS 候选、Agent 轨迹、JD 到速记问题、只读写操作和移动端溢出；本机 in-app browser 已完成同等路径的运行态检查，[首次绿色运行记录](https://github.com/Vergessen428/project_resume/actions/runs/29282001860)也已通过。后台任务取消仍以 API 回归测试为主。
5. **结果反馈不是招聘效果**：`outcome_source=self_reported` 只说明用户回填了通过/未通过/待定，样本门槛只控制描述性展示，不构成录用预测、评分因果或模型公平性结论。

本节是当前“能证明什么 / 不能证明什么”的硬边界，后续实现变化必须同步修改，不得只更新截图或营销文案。

## Goal 继续迭代

本轮里程碑已经覆盖可信诊断、来源边界和训练反馈的最小闭环。后续按下面顺序推进，不把尚未实现的能力写成当前卖点：

1. **研究 Agent 生产化**：继续为搜索回放、脚本壳、重定向阻断、空结果和网页提示注入补充固定夹具与限额测试；保持公开路由、不登录、不读 Cookie。
2. **质量评测**：把现有固定夹具扩展为可版本化的 relevance judgments、来源状态和诊断回归集；浏览器 smoke 已取得首次绿色 CI 记录，后续持续覆盖移动端文字折行和静态 Demo 的四条主路径。
3. **数据生命周期深化**：增加独立恢复演练、备份损坏提示和敏感信息提示的漏报评估；轮换与启动检查基础能力已经落地。
4. **存储升级**：只有在单用户 JSON 的规模、并发或查询能力成为瓶颈时，才先迁移 SQLite；多用户账号隔离和多实例场景再单独评估 Postgres、队列和对象存储。

当前仍明确不是：多用户 SaaS、录用预测、完整匿名化、小红书登录态爬虫或实时模拟面试状态机。备份损坏、恢复回滚失败、任务超时和取消已有临时目录故障演练并进入 CI；磁盘空间不足、进程中断和浏览器关键路径仍需后续演练。截图只能作为视觉证据，完整交互验收仍需浏览器 CI 可重复执行。
