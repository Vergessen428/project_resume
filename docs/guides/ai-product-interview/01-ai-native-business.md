# AI 产品经理面试准备一：大厂 AI Native 业务

## 适用岗位

这类岗位的核心不是给原有功能加一个模型，而是围绕 AI 能力重新设计用户任务和产品形态。典型场景包括 AI 助手、Agent、AI 搜索、智能创作、内容理解和多模态交互。

## 一、基础知识定义

### 1. AI Native 产品

AI Native 产品把模型的理解、生成、规划或执行能力作为核心产品能力，而不是附加按钮。产品流程、用户交互、评估指标和商业价值都需要围绕模型的不确定性重新设计。

### 2. Agent

Agent 是能够围绕目标理解任务、拆解步骤、调用工具并根据结果继续行动的系统。它与普通聊天的区别不在于“回答更长”，而在于能否完成一个可验证的任务。

### 3. Agent 的基本组成

通常包含任务目标、规划、上下文、Memory、Skill、Tool、模型判断、执行结果和人工接管。不是每个场景都需要完整 Agent，固定路径的任务用 Workflow 可能更可靠。

### 4. AI 产品的核心指标

不能只看模型准确率，还要看任务完成率、采纳率、响应时间、重复使用、人工接管率、bad case、成本和业务结果。不同业务要根据错误成本选择主指标和护栏指标。

### 5. AI 产品的不确定性

同样输入不一定得到完全相同的输出，模型还可能出现幻觉、误判、拒答和能力漂移。因此产品需要置信度、来源、人工确认、反馈、回滚和降级机制。

## 二、典型面试问答

### 1. 你如何判断一个 AI Native 产品的用户需求？

**回答框架：** 先看用户是否有明确任务，再看任务中是否存在理解、生成、搜索、规划或执行成本。AI 的价值应该体现在减少任务时间、提高结果质量或让原来无法完成的任务变得可行，而不是因为模型能生成就创建一个功能。

### 2. 一个 Agent 和普通聊天机器人有什么本质区别？

**回答框架：** 普通聊天主要输出信息，Agent 要围绕目标推进任务。判断标准是它是否能拆解步骤、调用工具、检查结果、处理失败并最终完成任务，而不是是否使用了“Agent”这个名字。

### 3. 如何判断一个场景适合做 Agent？

**回答框架：** 看任务是否需要多步判断、动态搜索、工具协同和上下文记忆，同时评估错误成本和人工接管成本。规则明确、路径固定的任务更适合普通自动化；需要动态决策但风险可控的任务才适合 Agent。

### 4. 如何设计 Agent 的人工接管机制？

**回答框架：** 先按风险和置信度分层，低风险任务自动处理，高风险、低置信度或不可逆操作转人工。人工接管时要展示用户目标、Agent 已完成步骤、依据和待确认动作，不能只把一堆原始日志交给人。

### 5. 如何评价一个 Agent 的好坏？

**回答框架：** 从四层评价：效果看任务完成和关键步骤正确率；体验看响应时间、交互成功和用户采纳；稳定性看 bad case、异常波动和工具失败；商业看成本、人工替代和转化。主指标要与任务目标绑定，不能用一个准确率覆盖所有场景。

### 6. Agent 输出很流畅，但用户并不采纳，说明什么？

**回答框架：** 可能是答案不可信、不可执行、缺少来源，或者没有嵌入用户实际流程。下一步要区分内容质量和产品体验：检查用户修改、拒绝、退出和后续任务结果，而不是继续优化文案流畅度。

### 7. 如果 Agent 的任务完成率突然下降，你如何排查？

**回答框架：** 先确认指标和统计口径，再分层排查输入分布、模型版本、Prompt、工具调用、数据源和业务环境。确认原因前先做降级、回滚或人工兜底，再决定补数据、改策略、调阈值还是换模型。

### 8. AI 产品如何处理模型幻觉？

**回答框架：** 高风险场景不能只依赖 Prompt。可以使用检索来源、引用展示、结构化输出、事实校验、置信度分层和人工确认；当无法验证时明确表达不确定，而不是强行给出答案。

### 9. RAG 和 Fine-tuning 应该如何选择？

**回答框架：** 知识经常更新、需要引用来源或希望快速维护时优先 RAG；需要模型稳定掌握格式、风格或行为模式时考虑 Fine-tuning。先判断问题是“模型不知道”还是“模型不会按要求做”，再选择方案。

### 10. 如何设计 AI 产品的反馈闭环？

**回答框架：** 收集采纳、修改、拒绝、重新生成、人工审核和最终任务结果，并区分模型错误、需求误解和用户偏好。反馈经过筛选后进入 bad case、评估集、Prompt 或产品策略，不能把所有点赞直接当训练数据。

### 11. AI Native 产品的 MVP 怎么做？

**回答框架：** 选择一个高频、边界清楚、可衡量的核心任务，先证明 AI 能显著改善一个环节。比如先验证“根据用户目标完成一次可追踪的商品比较”，而不是直接做覆盖搜索、推荐、售后和复购的万能 Agent。

### 12. AI 产品如何建立用户信任？

**回答框架：** 让用户看到依据、来源、置信度和下一步动作，并允许修改、拒绝和人工确认。信任来自可理解、可纠正和稳定，不是来自宣传模型参数更大。

### 13. 多模态什么时候有真实价值？

**回答框架：** 当任务信息天然分布在图片、视频、语音和文本中时，多模态能减少理解损失，例如内容审核、视频理解和图文创作。如果任务只需要文本，加入多模态只会增加成本和失败面。

### 14. 如何把 AI 能力转成业务价值？

**回答框架：** 先画清模型输出如何进入用户任务，再连接到效率、转化、留存或成本。不能只说模型准确率提升了多少，要解释用户因此完成了什么、业务因此改变了什么。

### 15. 大厂 AI Native 业务最容易踩什么坑？

**回答框架：** 一是只追求模型效果，忽略交互和任务完成；二是做万能 Agent，范围过大无法验证；三是忽略风险和人工兜底；四是把调用量当价值。优先做场景闭环和可观测评估，比堆功能更重要。

## 三、准备重点

- 能解释为什么这个场景需要 AI；
- 能把 Agent 拆成任务、工具、上下文、输出和人工接管；
- 能同时讲模型指标、用户指标和业务指标；
- 能说明 AI 的不确定性如何被产品化管理；
- 能从 MVP 和用户价值出发控制 Agent 范围。
